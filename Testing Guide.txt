Setup Guide
# 🚀 Complete Setup Guide - Scholarship Management Portal

## 📋 Prerequisites

Before starting, ensure you have the following installed:

- **Node.js** (v16 or higher) - [Download here](https://nodejs.org/)
- **PostgreSQL** (v12 or higher) - [Download here](https://www.postgresql.org/download/)
- **Git** - [Download here](https://git-scm.com/)
- **Code Editor** (VS Code recommended)

## 🗄️ Step 1: Database Setup

### 1.1 Install PostgreSQL
```bash
# Windows (using chocolatey)
choco install postgresql

# macOS (using homebrew)
brew install postgresql
brew services start postgresql

# Ubuntu/Debian
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
```

### 1.2 Create Database and User
```bash
# Connect to PostgreSQL as superuser
psql -U postgres

# Create database and user
CREATE DATABASE scholarship_management;
CREATE USER scholarship_user WITH PASSWORD 'scholarship_pass';
GRANT ALL PRIVILEGES ON DATABASE scholarship_management TO scholarship_user;

# Exit PostgreSQL
\q
```

### 1.3 Verify Database Connection
```bash
# Test connection
psql -U scholarship_user -d scholarship_management -h localhost
```

## 🔧 Step 2: Backend Setup

### 2.1 Navigate to Backend Directory
```bash
cd backend
```

### 2.2 Install Dependencies
```bash
npm install
```

### 2.3 Setup Environment Variables
```bash
# Copy environment template
cp .env.example .env
```

Edit the `.env` file with these values:
```env
# Server Configuration
NODE_ENV=development
PORT=3001
API_VERSION=v1

# Database Configuration
DB_HOST=localhost
DB_PORT=5432
DB_NAME=scholarship_management
DB_USER=scholarship_user
DB_PASSWORD=scholarship_pass
DB_DIALECT=postgres

# JWT Configuration
JWT_SECRET=your-super-secret-jwt-key-for-development-only
JWT_REFRESH_SECRET=your-super-secret-refresh-key-for-development-only
JWT_EXPIRE=24h
JWT_REFRESH_EXPIRE=7d

# Email Configuration (for development - use Ethereal Email)
SMTP_HOST=smtp.ethereal.email
SMTP_PORT=587
SMTP_USER=your-ethereal-user
SMTP_PASS=your-ethereal-pass
FROM_EMAIL=noreply@scholarportal.edu
FROM_NAME=ScholarPortal

# File Upload Configuration
UPLOAD_PATH=./uploads
MAX_FILE_SIZE=5242880
ALLOWED_FILE_TYPES=pdf,doc,docx,jpg,jpeg,png

# Security Configuration
BCRYPT_ROUNDS=12
RATE_LIMIT_WINDOW=15
RATE_LIMIT_MAX=100

# Frontend URL
FRONTEND_URL=http://localhost:3000

# Logging
LOG_LEVEL=info
LOG_FILE=logs/app.log
```

### 2.4 Create Database Schema
```bash
# Run the database schema creation
psql -U scholarship_user -d scholarship_management -h localhost -f ../database/schema/02-table-definitions.sql
psql -U scholarship_user -d scholarship_management -h localhost -f ../database/schema/03-application-workflow.sql
psql -U scholarship_user -d scholarship_management -h localhost -f ../database/schema/04-finance-notifications.sql
psql -U scholarship_user -d scholarship_management -h localhost -f ../database/schema/05-triggers-functions.sql
psql -U scholarship_user -d scholarship_management -h localhost -f ../database/schema/06-sample-data.sql
```

### 2.5 Create Upload Directory
```bash
mkdir -p uploads/documents
mkdir -p logs
```

### 2.6 Start Backend Server
```bash
npm run dev
```

You should see:
```
✅ Database connection established successfully.
🚀 Server running on port 3001
📚 API Documentation: http://localhost:3001/api/v1/health
🌍 Environment: development
```

## 🎨 Step 3: Frontend Setup

### 3.1 Open New Terminal and Navigate to Frontend

## 🔧 Troubleshooting

### Common Issues and Solutions

#### 1. Database Connection Error
```bash
# Check if PostgreSQL is running
sudo systemctl status postgresql  # Linux
brew services list | grep postgres  # macOS

# Restart PostgreSQL if needed
sudo systemctl restart postgresql  # Linux
brew services restart postgresql  # macOS
```

#### 2. Port Already in Use
```bash
# Kill process on port 3001 (backend)
npx kill-port 3001

# Kill process on port 3000 (frontend)
npx kill-port 3000
```

#### 3. Node Modules Issues
```bash
# Clear npm cache and reinstall
rm -rf node_modules package-lock.json
npm cache clean --force
npm install
```

#### 4. Database Schema Issues
```bash
# Drop and recreate database
psql -U postgres
DROP DATABASE scholarship_management;
CREATE DATABASE scholarship_management;
GRANT ALL PRIVILEGES ON DATABASE scholarship_management TO scholarship_user;
\q

# Re-run schema scripts
# (Follow Step 2.4 again)
```

```bash
# Connect to database
psql -U scholarship_user -d scholarship_management -h localhost

# List all tables
\dt

# View users
SELECT email, first_name, last_name, r.name as role 
FROM users u 
JOIN roles r ON u.role_id = r.id;

# View scholarships
SELECT name, amount, status, application_deadline 
FROM scholarships;

# View applications with details
SELECT 
    a.status,
    s.name as scholarship_name,
    u.first_name || ' ' || u.last_name as student_name,
    a.submitted_at
FROM applications a
JOIN scholarships s ON a.scholarship_id = s.id
JOIN students st ON a.student_id = st.id
JOIN users u ON st.user_id = u.id
ORDER BY a.created_at DESC;

# Exit database
\q
```

# 🔧 Troubleshooting Common Issues

### Issue 1: "psql: command not found"
**Windows:**
1. **Add PostgreSQL to PATH:**
   - Open System Properties → Environment Variables
   - Add to PATH: `C:\Program Files\PostgreSQL\15\bin`
   - Restart command prompt

**macOS:**
```bash
# Add to PATH
echo 'export PATH="/usr/local/opt/postgresql/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### Issue 2: "Connection refused" or "Authentication failed"
```bash
# Check if PostgreSQL is running
# Windows
sc query postgresql-x64-15

# macOS
brew services list | grep postgres

# Linux
sudo systemctl status postgresql

# Start PostgreSQL if not running
# Windows: Start from Services or
net start postgresql-x64-15

# macOS
brew services start postgresql

# Linux
sudo systemctl start postgresql
```

### Issue 3: "Database does not exist"
```bash
# Connect as postgres superuser and create database
psql -U postgres
CREATE DATABASE scholarship_management;
\q
```

### Issue 4: "Permission denied for schema public"
```bash
# Connect as postgres and grant permissions
psql -U postgres -d scholarship_management
GRANT ALL ON SCHEMA public TO scholarship_user;
GRANT CREATE ON SCHEMA public TO scholarship_user;
\q
```

### Issue 5: Backend crashes with "Cannot read properties of undefined"
This was the Sequelize Op issue - it's been fixed. If you still see it:
```bash
# Restart backend
cd backend
npm run dev
```

### Issue 6: Frontend won't start
```bash
# Clear cache and reinstall
npm cache clean --force
rmdir /s node_modules
del package-lock.json
npm install
npm start
```

### Issue 7: "Port already in use"
```bash
# Kill processes on ports
# Windows
netstat -ano | findstr :3000
netstat -ano | findstr :3001
# Note the PID and kill it:
taskkill /PID <PID> /F

# Or use npx
npx kill-port 3000
npx kill-port 3001
```

# Scholarship Management Backend API

A comprehensive, secure RESTful backend system for managing college-level scholarships, applications, and awards.

- **Approval System**: Multi-level approval workflow (Coordinator → Committee → Finance)

- **Notification System**: Email notifications for all major events
- **Audit Logging**: Complete audit trail for all system actions

### Security Features

- **Password Hashing** with bcrypt (12 rounds)
- **Rate Limiting** on all endpoints
- **Input Validation** with Joi schemas
- **SQL Injection Prevention** with Sequelize ORM

```javascript
// Joi schema validation
const schema = Joi.object({

  password: Joi.string().min(8).pattern(/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)/).required()
});
```

### Rate Limiting
```javascript
// Different limits for different endpoints
const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 5 // 5 attempts per window
});
```

### SQL Injection Prevention
```javascript
// Sequelize ORM with parameterized queries
const users = await User.findAll({
  where: {
    email: { [Op.iLike]: `%${searchTerm}%` }
  }
});
```
